## 1. Le principe : des clés, pas des comptes

WireGuard fonctionne comme SSH : chaque machine (*peer*) possède une **paire de clés**. On échange les clés *publiques*, et le tunnel s'établit dès que deux peers se reconnaissent. Pas de session, pas de négociation complexe.

<Note title="Bon à savoir">
  WireGuard vit dans le noyau Linux depuis la version 5.6 : aucune dépendance lourde à installer côté noyau récent.
</Note>

## 2. Générer les clés

```
$ wg genkey | tee privkey | wg pubkey > pubkey
$ cat privkey   # clé privée — à garder secrète
$ cat pubkey    # clé publique — à partager
```

## 3. Une configuration minimale

Le fichier `/etc/wireguard/wg0.conf` décrit l'interface locale et chaque peer distant :

```
[Interface]
PrivateKey = <clé privée locale>
Address    = 10.0.0.1/24
ListenPort = 51820

[Peer]
PublicKey  = <clé publique du pair>
AllowedIPs = 10.0.0.2/32
Endpoint   = vpn.exemple.org:51820
```

On active le tout avec `wg-quick up wg0`. Le champ `AllowedIPs` joue un double rôle : routage *et* filtrage des adresses autorisées dans le tunnel.

<Note type="warning">
  Mettre `AllowedIPs = 0.0.0.0/0` route **tout** le trafic dans le tunnel. Pratique pour un VPN « full », à éviter si vous ne vouliez qu'un accès à un sous-réseau.
</Note>
