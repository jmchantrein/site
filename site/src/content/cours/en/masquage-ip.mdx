## 1. Network or host: the boundary

An IPv4 address is 32 bits long. The CIDR **prefix** (the `/24` in `192.168.1.0/24`) indicates how many bits, starting from the left, identify the *network*. The remaining bits identify the *host* within that network.

<Note title="The subnet mask">
  The **subnet mask** is the address whose network bits are set to 1 and whose host bits are set to 0. `/24` ⇒ `255.255.255.0`.
</Note>

## 2. What the mask determines

- **Network address**: all host bits set to 0 — the identity of the network.
- **Broadcast**: all host bits set to 1 — reaches every host.
- **Usable hosts**: $2^{(32-p)} - 2$ (subtracting the network and broadcast addresses).

<Note type="warning" title="Edge cases">
  `/31` has no broadcast address (2 hosts, point-to-point link, RFC 3021) and `/32` designates a single host.
</Note>

<Slides title="Slideshow — key takeaways">
  <Slide>
    ### Prefix = boundary

    The `/p` sets the number of network bits. The larger `p` is, the smaller the network (fewer hosts).
  </Slide>
  <Slide>
    ### Network & broadcast

    Host bits at 0 ⇒ network address; host bits at 1 ⇒ broadcast. Neither can be assigned to a machine.
  </Slide>
  <Slide>
    ### Counting hosts

    $2^{(32-p)} - 2$. Example: `/26` ⇒ $2^{6}-2 = 62$ hosts.
  </Slide>
