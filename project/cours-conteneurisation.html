Ce qu'est vraiment un conteneurWhat a container really is

Un conteneur n'est ni une machine virtuelle, ni de la magie : c'est un processus Linux ordinaire, à qui le noyau ment sur ce qu'il peut voir. Deux mécanismes suffisent à comprendre l'essentiel.A container is neither a virtual machine nor magic: it is an ordinary Linux process the kernel lies to about what it can see. Two mechanisms explain almost everything.

Glisser la bordure pour déplacer la colonneDrag the border to move the column

1. Le partage du noyau1. Sharing the kernel

Une machine virtuelle embarque un noyau complet par-dessus un hyperviseur. Un conteneur, lui, partage le noyau de l'hôte : il n'y a qu'un seul noyau pour tous les conteneurs. C'est ce qui rend le démarrage quasi instantané et l'empreinte mémoire minuscule.A virtual machine ships a full kernel on top of a hypervisor. A container instead shares the host kernel: there is a single kernel for every container. That is what makes startup near-instant and the memory footprint tiny.

2. Les namespaces : mentir sur ce qu'on voit2. namespaces: lying about what is visible

Un namespace (espace de noms) cloisonne une ressource du noyau. Le namespace PID donne au conteneur sa propre numérotation de processus : à l'intérieur, votre application est le processus n° 1, et elle ne voit aucun autre processus de l'hôte.A namespace partitions a kernel resource. The PID namespace gives the container its own process numbering: inside, your application is process #1 and sees no other host process.

Un seul processus, numéroté 1. Le conteneur ne perçoit pas les centaines de processus de la machine hôte : le namespace PID les a masqués.A single process, numbered 1. The container cannot perceive the host's hundreds of processes: the PID namespace hid them.

3. Les cgroups : limiter ce qu'on consomme3. cgroups: limiting what is consumed

Là où les namespaces gèrent la visibilité, les control groups (cgroups) gèrent les quantités : CPU, mémoire, E/S. Le partage CPU est proportionnel à un poids. Pour un conteneur \(i\) de poids \(w_i\), sa part garantie de CPU sous contention vaut :Where namespaces govern visibility, control groups (cgroups) govern amounts: CPU, memory, I/O. CPU sharing is proportional to a weight. For a container \(i\) with weight \(w_i\), its guaranteed CPU share under contention is:

$$\text{part}_i \;=\; \frac{w_i}{\displaystyle\sum_{j} w_j}$$

Avec deux conteneurs de poids 100 et 300, le second reçoit \(\frac{300}{400}=75\%\) du CPU uniquement lorsque la machine est saturée ; sinon, chacun prend ce qui est libre.With two containers of weight 100 and 300, the second gets \(\frac{300}{400}=75\%\) of the CPU only when the machine is saturated; otherwise each takes whatever is free.

!

AttentionCaution

L'option --privileged et le drapeau --pid=host désactivent l'isolation. Pratiques pour le débogage, dangereux en production : un processus du conteneur peut alors agir sur tout l'hôte.The --privileged option and the --pid=host flag disable isolation. Handy for debugging, dangerous in production: a container process can then act on the entire host.

1. create

Le conteneur est préparé à partir d'une image : son système de fichiers en couches est monté, mais aucun processus ne tourne encore.The container is prepared from an image: its layered filesystem is mounted, but no process runs yet.

2. start / run

Le noyau crée les namespaces et cgroups, puis lance le processus n° 1 du conteneur. C'est l'état actif.The kernel creates the namespaces and cgroups, then launches the container's process #1. This is the running state.

3. stop & rm

À l'arrêt, le processus 1 reçoit SIGTERM. La couche écriture subsiste jusqu'au rm, qui détruit le conteneur (mais pas l'image).On stop, process 1 receives SIGTERM. The writable layer persists until rm, which destroys the container (but not the image).

On applique la formule de partage proportionnel :Apply the proportional-share formula:

$$\text{part}_1 = \frac{256}{256+768} = \frac{256}{1024} = \frac{1}{4} = 25\%$$

Le premier conteneur reçoit 25 % du cœur, le second 75 % — mais seulement sous contention. À vide, chacun peut atteindre 100 %.The first container gets 25% of the core, the second 75% — but only under contention. When idle, each can reach 100%.
