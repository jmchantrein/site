DevOps & IA

Faire entrer un modèle dans une chaîne d'intégration continue, c'est arbitrer entre reproductibilité, coût et sûreté. Cet article relève à la fois de l'administration système et des sciences du calcul — d'où ses deux thématiques.Bringing a model into a CI pipeline means balancing reproducibility, cost and safety. This article belongs to both system administration and computing science — hence its two topics.

1. Reproductibilité : figer l'environnement1. Reproducibility: pin the environment

Un modèle n'est reproductible que si son environnement l'est. On épingle les versions (bibliothèques, pilotes CUDA, poids du modèle) et on construit une image de conteneur immuable — exactement les principes vus dans le cours sur la conteneurisation.A model is reproducible only if its environment is. You pin versions (libraries, CUDA drivers, model weights) and build an immutable container image — exactly the principles from the containerization course.

✓

AstuceTip

Versionnez les poids du modèle comme des artefacts (registre dédié), pas dans le dépôt Git : ils sont volumineux et binaires.Version model weights as artifacts (a dedicated registry), not in the Git repo: they are large and binary.

2. Le coût, mis en équation2. Cost, as an equation

Une étape d'inférence dans la CI a un coût qu'on peut estimer. Pour \(n\) requêtes de test, un coût unitaire \(c\) par millier de tokens et \(t\) tokens par requête :An inference step in CI has a cost you can estimate. For \(n\) test requests, a unit cost \(c\) per thousand tokens and \(t\) tokens per request:

$$C \;=\; n \cdot t \cdot \frac{c}{1000}$$

Avec \(n=200\) tests, \(t=1500\) tokens et \(c=0{,}50\) €, chaque exécution coûte \(200 \cdot 1500 \cdot 0{,}0005 = 150\) € — d'où l'intérêt de ne lancer ces tests que sur les branches concernées.With \(n=200\) tests, \(t=1500\) tokens and \(c=0.50\) €, each run costs \(200 \cdot 1500 \cdot 0.0005 = 150\) € — hence running these tests only on relevant branches.

3. Garde-fous3. Guardrails

Un modèle est non déterministe : un test ne peut pas exiger une sortie exacte. On évalue plutôt des propriétés (format valide, absence de fuite de secret, score au-dessus d'un seuil) et on fixe un budget de coût par exécution.A model is non-deterministic: a test cannot demand an exact output. Instead, evaluate properties (valid format, no secret leak, score above a threshold) and set a per-run cost budget.

✕

PiègePitfall

Ne mettez jamais une clé d'API de modèle en clair dans la configuration CI : utilisez les secrets chiffrés du runner, et faites-les expirer.Never put a model API key in plaintext in the CI config: use the runner's encrypted secrets, and rotate them.
