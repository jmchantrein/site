WireGuard, le VPN moderne

Là où les VPN classiques empilent des milliers de lignes de code, WireGuard tient en quelques milliers seulement — et sa configuration en quelques lignes. Voici l'essentiel.Where classic VPNs pile up thousands of lines of code, WireGuard fits in a few thousand — and its configuration in a few lines. Here's the gist.

1. Le principe : des clés, pas des comptes1. The idea: keys, not accounts

WireGuard fonctionne comme SSH : chaque machine (peer) possède une paire de clés. On échange les clés publiques, et le tunnel s'établit dès que deux peers se reconnaissent. Pas de session, pas de négociation complexe.WireGuard works like SSH: each machine (peer) holds a key pair. You exchange public keys, and the tunnel forms as soon as two peers recognise each other. No session, no complex negotiation.

Bon à savoirGood to know

WireGuard vit dans le noyau Linux depuis la version 5.6 : aucune dépendance lourde à installer côté noyau récent.WireGuard ships in the Linux kernel since 5.6: no heavy dependency to install on a recent kernel.

2. Générer les clés2. Generate the keys

$ wg genkey | tee privkey | wg pubkey > pubkey
$ cat privkey   # clé privée — à garder secrète
$ cat pubkey    # clé publique — à partager

3. Une configuration minimale3. A minimal configuration

Le fichier /etc/wireguard/wg0.conf décrit l'interface locale et chaque peer distant :The file /etc/wireguard/wg0.conf describes the local interface and each remote peer:

[Interface]
PrivateKey = <clé privée locale>
Address    = 10.0.0.1/24
ListenPort = 51820

[Peer]
PublicKey  = <clé publique du pair>
AllowedIPs = 10.0.0.2/32
Endpoint   = vpn.exemple.org:51820

On active le tout avec wg-quick up wg0. Le champ AllowedIPs joue un double rôle : routage et filtrage des adresses autorisées dans le tunnel.Bring it up with wg-quick up wg0. The AllowedIPs field plays a double role: routing and filtering which addresses are allowed in the tunnel.

!

AttentionCaution

Mettre AllowedIPs = 0.0.0.0/0 route tout le trafic dans le tunnel. Pratique pour un VPN « full », à éviter si vous ne vouliez qu'un accès à un sous-réseau.Setting AllowedIPs = 0.0.0.0/0 routes all traffic through the tunnel. Handy for a “full” VPN, avoid it if you only wanted access to one subnet.
